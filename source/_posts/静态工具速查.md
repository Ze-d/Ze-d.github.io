下面给你一份“刷题向”的 **Java 常用静态工具类速查**（可以直接贴到你的笔记里）。我按：**类 → 高频方法 → 典型用法 → 易错点** 来写。

------

## 0. 你会经常用到的静态工具类清单

- `java.util.Arrays`
- `java.util.Collections`
- `java.lang.Math`
- `java.lang.Integer / Long / Double / Character`
- `java.lang.String`（虽然不是工具类，但静态/常用方法很多）
- `java.util.Objects`
- `java.util.Comparator`
- `java.util.stream.Collectors`（能用但刷题不必强依赖）
- `java.util.Optional`（刷题很少必需，了解即可）

------

## 1) Arrays（数组工具类）

**高频方法**

- 排序：`Arrays.sort(arr)`、`Arrays.sort(arr, l, r)`
  对对象数组：`Arrays.sort(arr, comparator)`
- 搜索：`Arrays.binarySearch(arr, key)`
- 拷贝：`Arrays.copyOf(arr, newLen)`、`Arrays.copyOfRange(arr, l, r)`
- 填充：`Arrays.fill(arr, val)`、`Arrays.fill(arr, l, r, val)`
- 转字符串：`Arrays.toString(arr)`、`Arrays.deepToString(arr2d)`
- 比较：`Arrays.equals(a,b)`、`Arrays.deepEquals(a2d,b2d)`

**典型用法**

```java
Arrays.sort(nums);
int idx = Arrays.binarySearch(nums, target);
int[] b = Arrays.copyOfRange(nums, 2, 5); // [2,5)
Arrays.fill(dp, -1);
```

**易错点**

- `binarySearch` 只对 **已排序** 数组有意义；找不到返回负数：`-(insertionPoint) - 1`
- `Arrays.sort(int[])` 不能传 comparator（只能对象数组 `Integer[]` 才能自定义）

------

## 2) Collections（集合工具类）

**高频方法**

- 排序：`Collections.sort(list)` 或 `list.sort(comparator)`
- 反转：`Collections.reverse(list)`
- 最大/最小：`Collections.max(list)` / `min(list)`
- 频次：`Collections.frequency(list, x)`
- 填充：`Collections.fill(list, val)`（注意 list 必须已有固定 size）
- 不可变包装：`Collections.unmodifiableList(list)`（刷题少用）
- 空集合：`Collections.emptyList()` / `emptyMap()`（偶尔用）

**典型用法**

```java
Collections.sort(list);
Collections.reverse(list);
int mx = Collections.max(list);
```

**易错点**

- `Collections.fill(list, x)` 不会扩容；list 需要已有元素（比如 `new ArrayList<>(Collections.nCopies(n, 0))`）

------

## 3) Math（数学工具类）

**高频方法**

- `Math.max(a,b)` / `Math.min(a,b)`
- `Math.abs(x)`（注意 `Math.abs(Integer.MIN_VALUE)` 仍是负数）
- 幂/开方：`Math.pow(a,b)`、`Math.sqrt(x)`
- 取整：
  - `Math.floor(x)`、`Math.ceil(x)`（double）
  - `Math.round(x)`（四舍五入，返回 long/int）
- 随机：`Math.random()`（刷题一般不用）

**刷题常见技巧**

- 取上整除（int）常用：`(a + b - 1) / b`（比用 `ceil` 更稳）
- 防溢出取 mid：`int mid = l + (r - l) / 2;`

------

## 4) Integer / Long（数值工具类）

**高频常量**

- `Integer.MAX_VALUE / MIN_VALUE`
- `Long.MAX_VALUE / MIN_VALUE`

**高频方法**

- 进制/解析：`Integer.parseInt(s)`、`Long.parseLong(s)`
- 转字符串：`Integer.toString(x)`、`String.valueOf(x)`
- 比较：`Integer.compare(a,b)`（避免 `a-b` 溢出）
- 位运算相关（刷题很常用）：
  - `Integer.bitCount(x)`：1 的个数
  - `Integer.highestOneBit(x)`：最高位 1
  - `Integer.lowestOneBit(x)`：最低位 1
  - `Integer.numberOfLeadingZeros(x)`
  - `Integer.numberOfTrailingZeros(x)`
- 进制转换：`Integer.toBinaryString(x)`（调试很爽）

**典型用法**

```java
int bits = Integer.bitCount(mask);
int cmp = Integer.compare(b, a); // 大根堆 comparator
```

------

## 5) Character（字符工具类）

**高频方法**

- 判断：`Character.isDigit(c)`、`isLetter(c)`、`isLetterOrDigit(c)`、`isWhitespace(c)`
- 大小写：`Character.toLowerCase(c)`、`toUpperCase(c)`

**典型用法**

```java
if (Character.isDigit(c)) ...
```

------

## 6) String（不是工具类，但刷题最常用）

**高频方法**

- 长度/访问：`length()`、`charAt(i)`
- 子串：`substring(l, r)`（[l, r)）
- 查找：`indexOf(ch/str)`、`startsWith`、`endsWith`
- 分割：`split(regex)`（注意是正则）
- 替换：`replace`、`replaceAll(regex, ...)`
- 比较：`equals`、`compareTo`（字典序）
- 连接：`+` / `StringBuilder`

**易错点**

- `split(".")` 是错的（`.` 是正则任意字符），要写 `split("\\.")`
- 字符串拼接在循环里别用 `+`，用 `StringBuilder`

------

## 7) Objects（空指针安全 & equals）

**高频方法**

- `Objects.equals(a,b)`：安全比较（处理 null）
- `Objects.requireNonNull(x)`：断言非空（刷题少用）
- `Objects.hash(...)`：生成 hash（刷题一般不用）

**典型用法**

```java
if (Objects.equals(a, b)) ...
```

------

## 8) Comparator（比较器工具）

**高频用法**

- `Comparator.naturalOrder()` / `reverseOrder()`
- `Comparator.comparingInt(x -> x[0])`
- 链式：`.thenComparingInt(...)`

**典型用法（二维数组排序）**

```java
Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));
```

**易错点**

- 不要写 `(a,b) -> a[0]-b[0]`，可能溢出；用 `Integer.compare(a[0], b[0])` 或 `comparingInt`

------

## 9) Collections.nCopies（初始化固定长度列表）

这个在 DP / 初始化 list 很好用：

```java
List<Integer> list = new ArrayList<>(Collections.nCopies(n, 0));
```

注意：`nCopies` 返回的是不可变视图，外面套一层 `new ArrayList<>(...)` 才能修改。

------

## 10) Stream / Collectors（了解即可）

刷题不是必须，但有时写起来很短：

```java
List<Integer> list = Arrays.stream(nums).boxed().collect(Collectors.toList());
```

一般不建议重度依赖（会影响手写熟练度和可控性）。

- 比较器：`Comparator.comparingInt`（别用减法）
