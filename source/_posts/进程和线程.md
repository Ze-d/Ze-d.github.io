**计算机需要解决两个问题，一个是资源如何进行分配和隔离，一个是代码如何高效执行和并发执行**。

## 1.核心定义和根本区别

1. **进程**： 操作系统进行资源分配的基本单位
2. **线程**：CPU进行调度和执行的基本单位

**进程的虚拟地址空间**

```text
┌──────────────────────────────────────────────┐ 高地址
│ 内核空间（用户态不可直接访问）               │
├──────────────────────────────────────────────┤
│ 映射区（mmap、动态库、共享内存等）           │
├──────────────────────────────────────────────┤
│ 线程3的用户栈                                │
├──────────────────────────────────────────────┤
│ 线程2的用户栈                                │
├──────────────────────────────────────────────┤
│ 线程1的用户栈（主线程栈）                    │
├──────────────────────────────────────────────┤
│                  空闲区                      │
├──────────────────────────────────────────────┤
│ 堆（Heap）                                   │
├──────────────────────────────────────────────┤
│ BSS 段（未初始化全局变量/静态变量）          │
├──────────────────────────────────────────────┤
│ 数据段（已初始化全局变量/静态变量）          │
├──────────────────────────────────────────────┤
│ 代码段（Text，程序机器指令）                 │
└──────────────────────────────────────────────┘ 低地址
```

**线程的虚拟地址空间**


```text
线程 T1
┌──────────────────────────────────────┐
│ 线程ID / 状态 / 优先级               │
│ （由内核中的线程控制块 TCB 管理）    │
├──────────────────────────────────────┤
│ 程序计数器 PC                        │
├──────────────────────────────────────┤
│ 寄存器上下文                         │
├──────────────────────────────────────┤
│ 用户栈                              │
│  ├─ 栈帧：funcC()                    │
│  ├─ 栈帧：funcB()                    │
│  ├─ 栈帧：funcA()                    │
│  └─ 栈帧：main()                     │
├──────────────────────────────────────┤
│ 内核栈                              │
│  └─ 内核态执行时的调用现场           │
└──────────────────────────────────────┘
```

线程会共享所属进程的地址空间。尤其是：代码段，**堆**，全局变量，静态变量。

## 2.内存空间和资源管理

1. **进程的独立空间：**每一个进程都有自己独立的虚拟地址空间。代码段，数据段、堆栈等都是独立的。
2. **线程的空间构成**：线程中：每个线程的栈（用户栈和内核栈）和程序计数器以及寄存器上下文（记录cpu的执行上下文）都是**私有的**。同时，同一个进程的线程共享该进程的代码段，数据段，堆区以及打开的文件等资源。

## 3.隔离和共享

1. 进程间隔离：由于虚拟地址空间独立，一个进程的使用和崩溃不会影响其他的进行，这属于在操作系统的层级的保护机制
2. 线程间共享：线程间共享进程内存，使得数据交换变得很方便，同时引发两个问题：健壮性：如果一个线程因为非法访问内存崩溃，会使得整个进程崩溃。共享也会引发**并发安全问题**。

## 4.上下文开销

1. 进程切换：开销大，需要保存和恢复cpu寄存器状态，还需要切换虚拟内存地址。
2. 线程切换：开销小，切换的时候需要保存和恢复少量的线程私有数据，不需要切换页表。

## 5.通信与同步

1. 进程间通信：由于空间隔离，需要借助操作系统提供的机制（管道，消息队列，socket等）来进行通信，相对复杂。
2. 线程间通信：由于共享同一块内存，直接读写全局变量或者堆上的数据即可。重点在于同步和互斥，防止数据被并发篡改。

## 6.总结

| **维度**       | **进程 (Process)**                | **线程 (Thread)**                              |
| -------------- | --------------------------------- | ---------------------------------------------- |
| **根本属性**   | 资源分配的基本单位                | CPU调度的基本单位                              |
| **内存空间**   | 相互独立，有各自的虚拟地址空间    | 共享所在进程的地址空间；但拥有独立的栈和寄存器 |
| **隔离与健壮** | 强隔离，一个崩溃不影响其他进程    | 无隔离，一个线程崩溃会导致整个进程崩溃         |
| **切换开销**   | 大（需切换页表等资源）            | 小（只切换少许上下文，无需切换页表）           |
| **通信方式**   | 需借助 IPC 机制（管道、Socket等） | 直接读写共享内存，但需要锁等同步机制           |
